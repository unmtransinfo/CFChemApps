Depict Help

Overview

This web app consists of a portal which uses basic HTML, CSS and JS for the user interface, and a Python framework (Django) for the backend.
The Django backend takes in input in the form of:

  1. MDL Mol File (.mol, .sdf)
  2. SMILES (files with delimiter-separated values, such as CSV or TSV)
The app then returns the structures of the different molecules, which is shown to the user using the interface. The images for the structures can be downloaded in JPEG, PNG, or SVG format (as specified). RDKit is being used to convert the different representations of the molecules to their respective structures.

Interface

  • The upload area can be used to upload files. Input can also be pasted into the text box.
  • The Help button shows this page.
  • The Demo button can be used to test run the portal with some pre-existing SMILE representations.
  • The Reset button can be used to reset any inputs for processing.
  • The Input Options section specifies the input format. If "SMILES" is selected, the additional options correspond to the arguments of SmilesMolSupplier from RDKit.
  • The Output Options section specifies the format of images, as well as which molecules from the input to visualize.
    • For details on the "Sanitize" option, see here.
    • For details on the "Kekulize" option, see here.
  • The SMARTS text area will highlight the given SMARTS pattern (where found).
    • The "Align SMARTS" option will align subsequent depictions to the first found substructure.
  • The Error Log text area reports errors encountered by RDKit when parsing input.
  • The Depict button will submit the input for processing using the given options.
